## Summary

Define the Model Context Protocol (MCP) surface for Flow, enabling AI agents (Claude, GPT, Gemini) to natively drive video editing operations through a structured tool and resource API.

## Motivation

Flow's core innovation is being "AI-native." MCP is the protocol that makes this real — agents don't write glue code, they call tools directly. No other video project has an MCP surface.

## Design

### MCP Server

`flow-server` exposes MCP at `/mcp`, alongside REST and WebSocket endpoints.

### Tools

Tools are the primary interaction model for agents.

#### Media Tools

| Tool | Input | Output |
|---|---|---|
| `flow.media.probe` | `path: string` | `MediaInfo` (codec, resolution, duration) |
| `flow.media.list` | `path: string` | `MediaInfo[]` |

#### Project Tools

| Tool | Input | Output |
|---|---|---|
| `flow.project.create` | `name: string` | `ProjectId` |
| `flow.project.list` | — | `ProjectInfo[]` |
| `flow.project.log` | `project_id: string` | `Action[]` |
| `flow.project.diff` | `project_id, ref_a, ref_b` | `Diff` |
| `flow.project.checkout` | `project_id, ref` | `Timeline` |
| `flow.project.branch` | `project_id, name` | `ProjectInfo` |

#### Timeline Tools

| Tool | Input | Output |
|---|---|---|
| `flow.timeline.get` | `project_id` | `Timeline` (OTIO JSON) |
| `flow.timeline.apply` | `project_id, action` | `Timeline` (after apply) |
| `flow.timeline.plan` | `project_id, action` | `Plan` (human-readable) |

#### Render Tools

| Tool | Input | Output |
|---|---|---|
| `flow.render.start` | `project_id, action` | `JobId` |
| `flow.render.status` | `job_id` | `JobStatus` |
| `flow.render.wait` | `job_id` | `JobResult` |
| `flow.render.artifacts` | `job_id` | `Artifact[]` |
| `flow.render.cancel` | `job_id` | — |

#### Effect Tools

| Tool | Input | Output |
|---|---|---|
| `flow.effects.list` | — | `EffectInfo[]` |
| `flow.effects.describe` | `effect_id` | `EffectSchema` |

### Resources

| URI | Returns |
|---|---|
| `flow://project/{id}/timeline` | OTIO JSON |
| `flow://project/{id}/timeline?ref={ref}` | OTIO JSON at checkpoint |
| `flow://project/{id}/media` | MediaReference[] |
| `flow://effects/{id}` | Effect schema + docs |
| `flow://presets/{id}` | Render preset spec |
| `flow://schemas/{schema_name}` | Raw JSON Schema |

### Progress Streaming

For long-running renders, `flow.render.start` returns a `JobId`. The agent can:
- Poll `flow.render.status(job_id)` for progress (0–100%)
- Subscribe to WebSocket at `WS /v1/jobs/:id/stream` for real-time updates
- Call `flow.render.wait(job_id)` (blocks until done, with timeout)

### Example Session

```
→ flow.project.create(name: "My Edit")
← { project_id: "proj_01HXZ..." }

→ flow.media.probe(path: "interview.mov")
← { codec: "h264", resolution: "3840x2160", duration: 842 }

→ flow.timeline.apply(project_id: "proj_01HXZ...", action: { op: "clip.add", ... })
← { timeline: { ... }, actions_applied: 1 }

→ flow.render.start(project_id: "proj_01HXZ...", action: { op: "render", ... })
← { job_id: "job_01HYA...", estimated_time: "37s" }

→ flow.render.wait(job_id: "job_01HYA...")
← { status: "completed", artifacts: ["out.mp4"] }
```

## Alternatives Considered

- **REST-only** — agents can use it, but need custom wrappers per LLM. Rejected.
- **gRPC-only** — not natively supported by current LLMs. Rejected.
- **CLI wrapper** — agent calls `flow run` — no streaming, no progress. Rejected.

## Open Questions

- Should non-render operations be synchronous MCP tools, or always async?
- How do we handle MCP authentication (API keys in the MCP handshake)?

## Implementation Plan

2. Implement tool handlers for all tools above
3. Implement resource handlers for all resources above
4. Wire WebSocket progress into MCP tool responses
